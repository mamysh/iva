# Эксплуатация на сервере

[English version](../deploy.md)

Обычная установка запускает на одном VPS агента, Telegram polling-мост и семь таймеров. Опциональный
userbot добавляет третий постоянный сервис, а уведомления об обновлениях — восьмой таймер, только после
явного включения.

## Telegram

Iva сама опрашивает Telegram, поэтому домен, публичный HTTPS, nginx и webhook не нужны. Сообщения идут
по порядку, а offset сдвигается только после успешной передачи агенту. Команды восстановления мост
обрабатывает отдельно, поэтому `/restart` и `/update` доступны даже при зависшем агенте.

## Сервисы и таймеры

| Unit | Назначение |
|---|---|
| `iva.service` | агент |
| `iva-telegram-poll.service` | Telegram long polling |
| `iva-memory-daily.timer` | дневная память в 04:00 |
| weekly/monthly/yearly timers | недельные, месячные и годовые свёртки |
| `iva-memory-doctor.timer` | проверка vault и git push в 05:00 |
| `iva-reminders.timer` | напоминания каждые 5 минут |
| `iva-observe.timer` | технические метрики раз в час |
| `iva-update-check.timer` | opt-in проверка `origin/main` ежедневно в 10:00 по `ASSISTANT_TIMEZONE` |

Время берётся из часового пояса сервера. Пропущенная во время выключения задача запускается после старта.

```bash
iva status
iva doctor
systemctl --user list-timers 'iva-*'
iva logs
iva logs poll
```

Не редактируйте `~/.config/systemd/user/iva-*` вручную: источником служит код Iva. Неизменённые units
при restart не перезаписываются, чтобы systemd не терял историю ночных задач.

## Обновления

`iva update` собирает и тестирует новую версию отдельно, затем переключает код только на время короткого
рестарта. Неудачная сборка не трогает рабочую версию, а неудачный readiness-check возвращает предыдущую.

`iva update-check on` включает ежедневное уведомление один раз на новый target commit. Кнопки
«Посмотреть / Обновить / Позже» используют существующий `/update` flow; таймер только сравнивает
`origin/main` и никогда не устанавливает код автоматически. Проверка использует общий update lock и
не пересекается с CLI/Telegram transaction. Отключение: `iva update-check off`.

## Workflow

По умолчанию состояние файловое. Для долгоживущей установки можно явно включить PostgreSQL 17:

```bash
iva workflow-postgres enable
```

Команда идемпотентна, проверяет схему и двумя короткими запросами доказывает сохранение сессии после
перезапуска. PostgreSQL остаётся opt-in. `iva recover` ремонтирует прерванную работу без удаления данных;
`iva reset` отменяет активные сессии после подтверждения, сохраняя историю, vault, задачи и напоминания.

## Перенос

Используйте `iva backup` и `iva restore`, остановив старый сервер до запуска нового. Подробности:
[data-and-backup.md](data-and-backup.md).
