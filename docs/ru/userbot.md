# Telegram userbot (beta, opt-in)

[English version](../userbot.md)

> **Beta и риск блокировки аккаунта.** Автоматизация личного Telegram может нарушать правила Telegram.
> Используйте отдельный тестовый аккаунт и не доверяйте функции критичные действия.

Userbot позволяет Iva читать личный Telegram-аккаунт через локальный MCP-прокси. Он выключен по
умолчанию, слушает только `127.0.0.1` и в поддерживаемом режиме предоставляет read-only инструменты.
Отправка и изменение данных не входят в безопасный default.

## Подключение

Безопаснее вводить `api_id` и `api_hash` на сервере, а не в переписке с моделью:

```bash
iva userbot creds
iva userbot setup
```

Затем напишите боту «подключи мой телеграм» и отсканируйте QR через **Настройки → Устройства →
Подключить устройство**. Если аккаунт использует 2FA, не передавайте основной пароль через чат;
используйте отдельный временный пароль и смените его после подключения.

```bash
iva userbot status
iva userbot off
```

Default `TELEGRAM_EXPOSED_TOOLS=read-only` оставляет 49 upstream read-only инструментов и четыре
инструмента подключения. Режим `all` открывает 116 upstream-инструментов, считается высоким риском и
не поддерживается как default. Сессия и proxy-token хранятся приватно на сервере; portable backup
копирует их, но никогда сам не включает userbot после восстановления.
