# Если что-то не работает

[English version](../troubleshooting.md)

Начинайте с одной команды:

```bash
iva doctor
```

`degraded` — Iva отвечает, но есть предупреждение. `blocked` — сначала выполните указанное действие.
Для безопасной передачи в support используйте `iva doctor --json`; он не содержит ключей, ID, URL базы,
приватных путей или памяти.

## Бот молчит или завис ответ

```bash
iva status
iva recover
```

Если Workflow недоступен, сначала восстановите диск/базу. `/restart` перезапускает только агент;
`/new` отменяет активную сессию. `iva reset` используйте только когда согласны потерять незавершённые
диалоги: завершённая история, vault, задачи и напоминания сохранятся.

## Обновление откатилось

`ROLLED BACK` означает, что staging-проверка или readiness не прошли и предыдущая версия восстановлена.
Запустите `iva doctor`. Не удаляйте `.iva-update/previous/` до диагностики. При `sequential update required`
сначала установите промежуточную версию; при `backup blocked` исправьте PostgreSQL dump tools или доступ к
Workflow-хранилищу.

## Не пришло ежедневное предложение обновиться

Выполните `iva update-check status`: feature и timer должны быть enabled. `iva update-check on`
исправит настройку и сразу сделает одну read-only проверку. Один target commit намеренно предлагается
только один раз; сбой remote или Telegram не мешает обычным ответам и повторится по следующему timer.

## Сборка завершилась с exit 137

Серверу не хватило RAM. Установщик обычно создаёт swap; если этого не произошло, освободите минимум
2,6 ГБ диска и добавьте swap по инструкции из английского раздела [Build killed](../troubleshooting.md#build-killed-exit-137).

## Изменилась модель, но Iva использует старую

```bash
iva restart
```

Настройки модели читаются при старте процесса. Для Ollama default: DeepSeek V4 Pro — текст,
MiniMax M3 — изображения.

## Предупреждение о диске или росте

Запустите `iva status`, затем `iva doctor`. Не удаляйте Workflow-таблицы или память автоматически;
сначала определите растущую категорию и сделайте проверенный backup.

## Vault не отправляется в GitHub

```bash
gh auth login
systemctl --user start iva-memory-doctor.service
```

Не прикладывайте приватные логи или vault к публичному issue. При больших файлах сначала сделайте
локальный `git bundle`, затем исправляйте только неопубликованную историю без force-push памяти вслепую.

## Перенос или восстановление

Используйте полный [portable backup](data-and-backup.md). Один репозиторий vault не возвращает задачи,
напоминания, OAuth и Workflow-сессии. После restore не запускайте новый сервер, пока старый polling не остановлен.
