# Команды Iva

[English version](../cli.md)

## В Telegram

| Команда | Что делает |
|---|---|
| `/help` | список команд |
| `/task <текст>` / `/tasks` | добавить или показать задачи |
| `/reminders` | активные напоминания без вызова модели |
| `/digest` | утренний дайджест |
| `/restart` | перезапустить агент, сохранив Workflow |
| `/new`, `/clear`, `/compact` | отменить активные сессии и начать новый диалог |
| `/update` | предложить проверенное обновление кнопкой |
| `/model` | показать и раздельно выбрать text- и vision-провайдер/модель кнопками |
| `/think` | выбрать глубину рассуждения текста, если провайдер её поддерживает (сейчас Codex) |
| `/usage today|week|month|by-model|by-source` | расход токенов без вызова модели |

Мост выполняет служебные команды вне агента и отвечает только ID из allowlist. `/model` показывает
только провайдеры с уже настроенным ключом или Codex OAuth; ключи и произвольные model ID через
Telegram не принимаются. Text и vision меняются независимо. Перед применением выполняется ограниченная
синтетическая проверка, затем атомарно меняются только разрешённые ключи `.env`, перезапускается лишь
`iva.service` и проверяется readiness. При неуспехе предыдущий `.env` восстанавливается. Callback
привязан к владельцу и чату и истекает через пять минут.

## На сервере

| Команда | Что делает |
|---|---|
| `iva status` | короткое состояние и использование ресурсов |
| `iva doctor [--json]` | диагностика; JSON очищен от секретов |
| `iva update` | транзакционное обновление с автоматическим rollback |
| `iva backup [каталог]` | проверенный portable backup |
| `iva restore <каталог> --yes` | восстановление; сервисы остаются остановленными |
| `iva restart` | перезапуск процессов без удаления состояния |
| `iva recover` | восстановление прерванных Workflow-шагов |
| `iva reset` | отмена активных сессий после подтверждения |
| `iva config` | мастер настройки |
| `iva login` | вход через подписку ChatGPT |
| `iva workflow-postgres enable` | включить opt-in PostgreSQL |
| `iva reminders` / `iva usage week` | напоминания и статистика |
| `iva logs [poll\|reminders]` | технические логи выбранного сервиса |
| `iva start` / `iva stop` | запустить или остановить Iva |
| `iva version` | версия и commit |

`iva doctor` возвращает `healthy`, `degraded` или `blocked`. Только `blocked` означает, что ответы
ненадёжны. Не публикуйте обычные production-логи: для поддержки используйте очищенный `--json`.
