# Данные, backup, восстановление и перенос

[English version](../data-and-backup.md)

## Что сохраняется

Полный portable backup включает приватные настройки, vault с вложениями, задачи, напоминания, OAuth,
Telegram offset, опциональную userbot-сессию и активное Workflow-хранилище. Производные индексы, сборка,
зависимости и health-метрики не копируются — они пересоздаются.

Один только git-репозиторий vault восстанавливает память, но не задачи, напоминания, OAuth, позицию
Telegram polling или Workflow-сессии.

## Создание

```bash
iva backup
# или вне каталога кода:
iva backup /secure/off-host/iva-backup
```

Iva ненадолго останавливает все процессы записи, делает согласованный local/PostgreSQL snapshot,
записывает размеры и SHA-256, проверяет результат и возвращает ранее активные сервисы. Каталоги имеют
права `0700`, файлы — `0600`. Это защищает от других пользователей, но не является шифрованием.

Храните минимум две проверенные копии в разных местах. Ночной git push vault полезен, но не заменяет
полный backup.

## Восстановление

Установите совместимую версию на новом сервере, но не запускайте Telegram polling, затем:

```bash
iva restore /secure/iva-backup --yes
```

Проверяются права и контрольные суммы, затем восстанавливаются настройки, данные, vault и Workflow.
Сервисы специально остаются остановленными, чтобы два сервера не использовали один bot token.

После окончательного отключения старого сервера:

```bash
iva start
iva doctor
```

Для PostgreSQL целевая база должна быть отдельной и доступной. Никогда не направляйте restore в чужую
или общую базу. При ошибке оставьте сервисы остановленными, исправьте указанную причину и повторите restore.
