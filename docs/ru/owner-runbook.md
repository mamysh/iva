# Короткая инструкция владельца

[English version](../owner-runbook.md)

Команды выполняются внутри каталога Iva от пользователя, которому принадлежат сервисы.

## Установка

До stable используйте только закреплённую RC-команду из [install.md](install.md). Один bot token нельзя
одновременно запускать на двух серверах.

## Проверка

```bash
iva status
iva doctor
systemctl --user list-timers 'iva-*'
```

`degraded` означает неблокирующее предупреждение; `blocked` — ответы ненадёжны и сначала нужно выполнить
указанное действие восстановления. `iva doctor --json` создаёт очищенный отчёт без секретов.

## Обновление и восстановление

```bash
iva update
iva doctor
iva recover   # только когда советует doctor
```

Обновление сначала собирается и тестируется отдельно. Ошибка до активации не трогает рабочую версию,
ошибка readiness автоматически возвращает предыдущую. Не удаляйте таблицы PostgreSQL или vault вручную.

## Перенос сервера

1. На старом сервере выполните `iva backup <private-directory>`.
2. Остановите Iva на старом сервере.
3. Установите ту же версию на новом сервере.
4. Передайте backup защищённым способом и выполните `iva restore <directory> --yes`.
5. Убедитесь, что старый сервер выключен, затем выполните `iva start` и `iva doctor`.

Полная граница backup: [data-and-backup.md](data-and-backup.md).
