# Как помочь проекту

[English version](CONTRIBUTING.md)

Спасибо за вклад в Iva. Для изменений установки, данных, провайдеров или публичного поведения сначала
создайте issue. Делайте небольшие PR с одной понятной целью.

1. Сделайте fork и ветку от `main`.
2. Никогда не коммитьте `.env`, ключи, приватные логи, vault, дампы или production-данные.
3. Сначала запустите узкий тест, затем `npm run verify:pr`.
4. При изменении поведения обновите пользовательские инструкции на русском и английском.
5. В PR опишите влияние на пользователя, риски и проведённые проверки.

Карта кода и правила тестов: [CODEBASE_MAP.md](CODEBASE_MAP.md) и [docs/testing.md](docs/testing.md).
Расширения: [docs/extending.md](docs/extending.md). Уязвимости отправляйте по [SECURITY.ru.md](SECURITY.ru.md),
а не в публичные issues.
