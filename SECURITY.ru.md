# Политика безопасности

[English version](SECURITY.md)

## Поддерживаемые версии

Исправления безопасности выпускаются для последнего release candidate, а после `v0.3.0` — для
последней стабильной версии. Telegram userbot остаётся opt-in beta и не входит в stable-гарантии.

## Как сообщить об уязвимости

Не создавайте публичный issue с ключами, токенами, приватными логами, памятью или деталями эксплуатации.
Используйте приватную кнопку GitHub **Report a vulnerability**. Укажите версию, влияние и минимальное
воспроизведение без личных данных. Если private advisory недоступен, создайте публичный issue только с
просьбой дать приватный канал связи.

Никогда не прикладывайте `.env`, OAuth-файлы, SSH-ключи, URL базы, backup или production-логи.

Границы доверия описаны в [docs/ru/security.md](docs/ru/security.md).
